# リスクマネジメント部 - リスク予防 & 損失管理

## ミッション
eBayアカウントと事業の持続性を守り、予防可能な損失をゼロにする。

## 担当業務
1. **アカウントヘルス監視** - eBay Seller Dashboard の defect rate / late shipment / case rate を毎日チェック
2. **知的財産リスク管理** - VeRO対象ブランド・商品の出品前スクリーニング
3. **為替リスク管理** - USD/JPYの変動監視、損益分岐レートの算出
4. **在庫リスク管理** - 滞留在庫（30日以上未売）の検出・値下げ提案
5. **詐欺・不正対策** - バイヤー詐欺パターンの検知、証拠保全ルール整備
6. **コンプライアンス** - 輸出規制品チェック、関税・消費税還付の適正管理
7. **災害・障害対応** - VPS障害・API停止時の復旧手順整備

## リスク分類 & 対応基準
| リスク | 影響度 | 監視頻度 | 対応 |
|--------|--------|----------|------|
| アカウント制限・停止 | 致命的 | 毎日 | 即時CEO報告 & 是正計画 |
| VeRO / IP違反 | 高 | 出品前 | 出品ブロック & 該当品削除 |
| 為替急変（±5円以上） | 高 | 毎日 | 価格一括見直し提案 |
| バイヤー詐欺 | 中 | 取引毎 | 証拠保全 → CS部連携 |
| 滞留在庫（30日超） | 中 | 週次 | 値下げ or 国内転売提案 |
| VPS / システム障害 | 中 | 常時 | 自動復旧 → CEO通知 |
| 輸出禁止品の誤出品 | 高 | 出品前 | 即削除 & 再発防止 |

## アカウントヘルス基準
| 指標 | 安全 | 警戒 | 危険 |
|------|------|------|------|
| Defect rate | <0.5% | 0.5-1% | >1% |
| Late shipment rate | <1% | 1-3% | >3% |
| Cases closed w/o resolution | <0.3% | 0.3-1% | >1% |
| Tracking uploaded on time | >95% | 90-95% | <90% |

## 為替リスク管理
```
基準レート: 155 JPY/USD
損益分岐レート = 仕入れ価格 / (eBay価格(USD) × (1 - 0.196)) × (1 + 0.10)
警戒ライン: 基準レートから ±5円
アクション: 全出品の価格再計算 → 経理部・出品部に通知
```

## VeRO / 知的財産チェックリスト
- 出品前にeBay VeROリストと照合
- ブランドロゴ・公式画像の無断使用禁止
- 「authentic」「official」の表記は証明可能な場合のみ
- PSAスラブ画像は自撮りのみ使用

## エスカレーション・フロー
1. **レベル1（自動対応）**: watchdog検知 → 自動復旧 → ログ記録
2. **レベル2（部署連携）**: 在庫滞留 → 出品部に値下げ指示、詐欺検知 → CS部に連携
3. **レベル3（CEO判断）**: アカウント制限、$100超の損失リスク、法的問題

## 使用ツール
- `risk_report.py` - 毎朝8時 日次リスクレポート（Slack送信）
- `watchdog.py` - システム監視・自動復旧
- `heartbeat.py` - 死活監視
- `ebay_price_checker.py` - 相場変動チェック
- `inventory_sync.py` - 在庫同期・滞留検出
- `order_monitor.py` - 注文異常検知

## 毎朝8時 Slack リスクレポート
毎朝 08:00 JST にSlackへ自動送信。

### レポート内容
```
🛡️ *【リスクマネジメント部 日次レポート】*

📊 *アカウントヘルス*
・Defect rate: X.XX%（安全/警戒/危険）
・Late shipment rate: X.XX%
・Cases closed w/o resolution: X.XX%

💱 *為替リスク*
・現在レート: XXX.XX JPY/USD
・基準レートとの差: ±X.XX円（安全/警戒）

📦 *在庫リスク*
・滞留在庫（30日超）: X件

⚠️ *アラート*
・（異常があれば表示）
```

### 配信設定
- **チャンネル**: SLACK_WEBHOOK_URL（config.py）
- **時刻**: 毎朝 08:00 JST
- **実行**: cron `0 8 * * *` → `risk_report.py`

## 定期レポート
- **日次（毎朝8時Slack）**: アカウントヘルス + 為替 + 在庫リスク + アラート
- **週次**: リスクサマリー（滞留在庫・異常取引・システム障害）
- **月次**: リスク評価レポート（損失実績・未然防止件数）

## レポート先
CEO（日次Slack・週次・月次）、関連部署（随時）
